<< BBN: Bilateral-Branch Network with Cumulative Learning for Long-Tailed Visual Recognition >>论文阅读笔记

论文地址：“BBN: Bilateral-Branch Network with Cumulative Learning for Long-Tailed Visual Recognition”
复现代码：https://github.com/Megvii-Nanjing/BBN
核心思想：构造混合样本数据，解决长尾问题

概要

作者提出了一种双边分支网络（BBN: Bilateral-Branch Network），和一种训练策略（Cumulative Learning）解决计算机视觉的长尾问题（Long-Tailed Visual Recognition）。题目字字是重点系列！

作者在论文里详细的说明和分析了长尾问题的解决策略，和现有解决方法存在的问题，并基于现在的一些策略做了详细的对比实验，并抛出了他的观点，提出了他的模型。
个人感觉，作者写作的思路特别自然，首先先说长尾问题的解决方法：类别重平衡（class re-balancing）。具体来说，实现方式有两种，重采样（re-sampling）和重加权（re-weighting）。
然后，他对 class re-balancing 方法对解决长尾问题方法进行了探究，并提出了他的解决方式。具体来说，作者认为，re-balancing策略有效是因为，他可以通过让模型更加专注于尾部训练集并使训练数据集的分布和测试集的分布更像，但这种方法，可能会直接改变原数据集的分布，反而损失了直接表达整体特征的能力。其实，想说的是（抛出他的观点），在做re-balancing的时候最好先学习整体的数据特征，然后再根据长尾的数据特征，对model进行调整。这里，他做了一个对比实验，把model分为两部分：特征提取和分类器。实验的结论是，作者认为model在特征提取模块应该尽可能多的学习整体数据分布的特征，对分类器应该去学习更多的长尾分布数据的特征。

接下来，正式提出他的model，个人感觉主要贡献在两个

  1. 提出了一个基于双分支的负样本数据采样方法（可以认为是一种数据增强，他相当于构建了一个混合标签mixup）
  2. 提出了一种训练方式（前期专注于整体数据的特征，后期专注于长尾数据特征）

模型设计


作者设计了一个双分支网络，如图所示，模型有两个输入，一个输入正常数据（Uniform Sampler），一个输入经过重平衡后的样本数据（Reversed Sampler），将这两部分数据都输入到model中。model看图上来说也是两个，前半部分共享参数。然后把model的输出通过α结合，输出到分类器中。

Reversed Sampler采样方法：

  1. 统计每个类别的样本数量，然后用最多类别的样本数量处以每个类别的样本数量，归一化，作为每个类别的采样概率P（这样，样本数量多的数据，概率小）
  2. 根据概率P随机的抽取一个类别
  3. 在这个类别随机抽一个样本

α的计算公式：
α = 1 - (t/T)^2
这里t指当前epoch，T指总epoch。
这里思考一下，在第一个个epoch，α = 1，只训练正样本，这个时候的Loss，也只算了正样本的Loss。随着epoch迭代，模型越来越关注负样本，到最后一个epoch只训练负样本。达到了首先关注整体特征，后关注长尾数据特征的效果。

实验结果

作者做的对比实验里，个人感觉有两个对比实验比较有意思。
第一个是Reversed Sampler采样方法的对比实验，作者在论文里做了两个对比实验，一个是Uniform Sampler（和正样本相同），一个是Balanced Samper（按照类别数量正向采样）。这两个对比，相当于做了负采样的有效性对比实验，实验表明负采样有效。 这里，比较有意思的一个点是，Uniform 和 Balanced 的效果都没有两阶段训练的效果好，说明对长尾样本数据的增强还是显著有效的。 而，Balanced的Samper，数据融合以后，先当于做了一个Mixup，比单纯的mixup效果好一点。
第二个实验是做了α的公式的对比实验。里面有一个公式是 (t/T)^2，这样模型就先看负样本数据，后看正样本数据，效果显著性降低。证明先训练正样本数据（学习整体数据分布特征）再训练负样本数据（学习长尾数据特征）策略的有效性。此外，作者也做了两阶段训练，做对比。个人感觉他的这个训练方法是更加平滑的两端训练，实验结果表明这两种方法效果差不多。
这两个实验，只做一个模块，结果都是21多（错误率），都不做是24多，都做是17多。挺容易理解的，毕竟，只有两个都做，才达到了一开始学习正样本，后期学习负样本的效果。
